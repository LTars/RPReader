## 1. Форматтер текста (docx)

### Задача
Отформатировать ~46к токенов текста чата по правилам и получить .docx файл.

### Правила форматирования
- Содержание слово в слово, никаких правок текста
- Лишние разрывы строк убрать
- [Tars] и [zaveta] — имена авторов в квадратных скобках → вынести в подзаголовок
- zaveta везде → zaveta-Даша
- Несколько блоков подряд от одного автора — дата только у первого
- Даты и время удалить везде кроме строк с именем автора
- Звёздочки (*текст*) — оставить, bold + heading level 1
- Абзацы начинать с красной строки
- Имена персонажей — гиперлинки на другую вкладку документа (anchor: char_ИМЯ)
- zaveta-Даша в заголовке: имя болдом, дата/время курсивом
- Не менять тире, не исправлять ошибки

### Архитектура форматтера
**Проблемы первых версий:**
- JSON формат ответа раздувал выход в 3-4 раза → обрезание на 8к токенов
- docx CDN не загружался в iframe артефакта

**Финальная архитектура:**
1. Калибровочный чанк ~4000 символов → считаем коэффициент раздутия выхода
2. Вычисляем целевой размер чанка: `TARGET_OUT_TOKENS * charsPerOutTok / calibCoeff`
3. Режем остаток по точкам назад от целевого размера
4. Чанки последовательно → превью после каждого → ждём accept пользователя
5. Детект обрезания: `output_tokens >= 7995` → находим границу в источнике → пересчитываем нарезку
6. Retry 3 раза на rate limit / network ошибках
7. Фатальная ошибка → пауза с кнопкой "скачать что есть"
8. Финал → сборка docx, если упала → отдаём чанки .txt по отдельности

**Формат тегов в ответе модели (plain text):**
- `§A§ИМЯ§ДАТА§` — заголовок автора
- `§S§текст§` — звёздочки/разделитель
- `§P§текст§` — обычный абзац
- `§D§текст§` — реплика диалога
- `§C§ИМЯ§` — имя персонажа (→ InternalHyperlink в docx)
- `§E§` — пустая строка

**Статистика и стоимость:**
- Sonnet: $3.00/1M input, $15.00/1M output
- Лог показывает токены вверх/вниз и стоимость в реальном времени

**Файл:** `formatter.html` — самодостаточный HTML с встроенным JS, использует Anthropic API и docx CDN

## 2. Читалка для ролевого текста (RPReader)

### Задача
Веб-читалка для ролевого текста в формате чата, с персонажами, ссылками, поиском.

### Полный план фич

**Формат отображения:**
- Чат с пузырями и аватарками как в Телеграме
- Неавторизованный: первый пост справа, дальше поочерёдно
- Авторизованный автор: свои посты справа, чужие слева
- Аватарки — плейсхолдеры (пустые ссылки), потом реальные
- Цветовые метки авторов — консистентно везде

**Персонажи:**
- Отдельные JSON файлы на каждого, индекс в index.json
- Hover через 2 сек — всплывашка с именами и алиасами
- Клик → панель справа (десктоп) / bottom sheet (мобиль)
- Bottom sheet: смахнуть или раскрыть в fullscreen
- Fullscreen = отдельный URL `/character.html?id=...`
- При возврате из fullscreen — анимация подсветки строки как цитата в телеграме
- Страница персонажа: галерея, связи, список сцен где появляется
- Связи на старте — руками в JSON, потом автопоиск упоминаний

**Навигация:**
- Левая панель — тонкая полоска с иконками, разворачивается
- На мобиле — нижняя панель
- Поиск всегда в доступе, с фильтрами по контексту (текст / персонажи / комментарии)
- Прогресс чтения + заряд батареи в статус баре
- Таймлайн сцен (идея на будущее)

**Комментарии:**
- Стиль Google Docs — выделил текст → иконка → комментарий / вопрос / правка / закладка
- Авторизованный может скрыть от гостей
- Сохраняются в базе
- Модерация — отдельный интерфейс, визуально работает сразу с пометкой "на модерации"

**Закладки:**
- Авторизованные — в базе, везде
- Гости — в localStorage (привязаны к браузеру)
- Список в интерфейсе

**Реакции:**
- Эмодзи на строки и абзацы
- Только для авторизованных
- Отключаемые

**Режимы:**
- Режим чтения — UI прячется, ссылки и заметки форматируются под текст
- Режим редактирования — только whitelist, редактирование текста, ответы на комменты

**Обновление контента:**
- Вставил текст в браузере или переслал телеграм-боту
- Обработчик форматирует (регулярки + список имён)
- Показывает превью → после аппрува → коммит через GitHub API
- Добавляется в конец файла, контроль версий через git

**Дополнительно:**
- Офлайн через Service Worker
- Экспорт диапазона в PDF/MD (только авторизованным)
- Статистика чтения — личная и агрегированная
- Фильтр синего света, кнопка для дальтоников
- Связанные сцены на странице персонажа

**Авторизация:**
- Два уровня: ты и Даша (whitelist, можно добавлять через интерфейс)
- Загрузка картинок через браузер без кода
- Бэкенд — Supabase (на будущее, МВП без бэкенда)

### МВП — реализовано

**Файловая структура:**
```
RPReader/
├── index.html
├── reader.html
├── character.html
├── css/
│   ├── style.scss    ← редактировать здесь
│   └── style.css     ← скомпилированный
├── js/
│   ├── parser.js     ← парсер, правила из JSON
│   ├── characters.js ← тултип, панель, загрузка по требованию
│   ├── search.js     ← поиск
│   └── reader.js     ← оркестратор
├── content/
│   └── main.md       ← текст сюда
├── data/
│   ├── parser-rules.json
│   └── characters/
│       ├── index.json
│       └── example_char.json
└── .github/workflows/deploy.yml
```

**Парсер:**
- Правила подгружаются из `parser-rules.json` — можно добавлять авторов без правки кода
- Паттерны, трансформы, cleanup — всё конфигурируемо
- Блоки: `message | divider | empty`

**Персонаж JSON (шаблон):**
```json
{
  "id": "char_id",
  "names": ["Имя", "Прозвище", "Титул"],
  "description": "",
  "avatar": "",
  "relations": [{"characterId": "other", "type": "союзник", "note": ""}],
  "scenes": [{"id": "scene_001", "anchor": "scene-001", "title": "", "note": ""}],
  "gallery": []
}
```

**CSS:**
- SCSS с переменными, тёмная тема в стиле Claude
- GitHub Actions компилирует SCSS при пуше
- Переменные: `--bg`, `--surface`, `--border`, `--accent`, `--author-right`, `--author-left` и др.

**Деплой:**
- GitHub Pages, deploy from branch (main)
- Сайт: `https://ltars.github.io/RPReader/`
